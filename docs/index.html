Auth
Authentication via @convex-dev/auth with a Password provider. Users must be pre-created by an admin (users.addToCompany) before they can sign in. Use the exported signIn / signOut mutations from convex/auth.ts.
mutation

auth.signIn

Signs a user in with their email and staffId as the password. On success the session is established and subsequent queries/mutations run as that user.

Arguments
NameTypeRequiredDescription
providerstringrequiredMust be "password"
params.flowstringrequiredMust be "signIn" for existing users
params.emailstringrequiredUser's email address
params.passwordstringrequiredThe user's staffId (generated by admin on account creation)
Body / Arguments
{
  "provider": "password",
  "params": {
    "flow": "signIn",
    "email": "",
    "password": ""
  }
}
Example
import { useAuthActions } from "@convex-dev/auth/react";

const { signIn } = useAuthActions();
await signIn("password", { email: "jane@acme.com", password: staffId });
mutation

auth.signOut

Ends the current session and clears the auth token. No arguments required.

No arguments
Example
import { useAuthActions } from "@convex-dev/auth/react";

const { signOut } = useAuthActions();
await signOut();
